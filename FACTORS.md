# M1 Factor Definitions (clear + static)

M1 is a **simple, static, linear** directional model. It combines factor GROUPS by
fixed ratios. Anything dynamic / tuned (risk-weighting, transaction-cost thresholds)
lives in a SEPARATE layer, not in M1.

## Decision: merge in M1, but keep them SEPARATE for M2

This reconciles the mentor's two comments:
- *"momentum and trend are very close → one category"* (Xuesong) → in **M1** they are
  **merged** into one static `technical` group (50/50 of each z-score). Keeping them
  as two linear coefficients would split one signal and blur attribution. M1 stays
  simple.
- *"for M2 ... figure during which period which individual factor does better — that's
  where meta-labeling kicks in"* → **M2 receives momentum, trend, and macro
  SEPARATELY** (not the M1 merge) so it can **dynamically time them by regime**. M2
  can only weight momentum vs trend if it sees them un-merged.

So: **M1 = merged & static; M2 = individual & dynamic.** The 50/50 M1 split is a
config choice and reversible.

## Factor definition — M1 (STATIC → only momentum + trend)

Principle: **static factors → M1, dynamic factors → M2.** Momentum and trend are the
only static price signals, so M1 = technical alone.

| M1 factor | Weight | Definition (no look-ahead: every input is `shift(1)`) |
|---|---:|---|
| **Technical** | 1.0 | `0.5·z(momentum) + 0.5·z(trend)`. Momentum = avg of `pct_change(w)`, w∈{4,12,26,52} wk. Trend = `MA10/MA40 − 1` (ratio → price-level independent). Each z-scored across assets each week. |

M1 score = the technical score. Assets compete on one cross-sectional axis.

## Dynamic factors → M2 (NOT in M1)
- **Macro** — asset-class tilt + regime features (growth, inflation, curve, credit, VIX).
  Dynamic / regime-varying → an M2 feature, used to regime-date the meta-label.
- **Risk (volatility)** — vol / asset-quality. Time-varying → an M2 feature (dynamic
  risk-weighting), not a static M1 overlay. (`risk_layer.enabled: false`.)
- **Transaction-cost thresholds / tuning** — deferred to a later layer (not built now).
- M2 (logistic regression) sees momentum, trend, macro, vol SEPARATELY + regime, and
  dynamically decides which to trust per period — "that's where meta-labeling kicks in."

## Worked example — 3-asset ranking (Xuesong template)

Real numbers, universe = SPY / TLT / GLD, generated by `examples/worked_example.py`
(date 2026-06-19):

| ticker | mom_12w | trend_10_40 | z_mom | z_trend | **technical** | rank | tilt vs bench | **weight** |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| SPY | 0.144 | 0.062 | 1.078 | 0.990 | **1.034** | 3 | +0.100 | **0.433** |
| GLD | −0.065 | 0.026 | −0.898 | 0.019 | −0.439 | 2 | 0.000 | 0.333 |
| TLT | 0.011 | −0.012 | −0.179 | −1.010 | −0.594 | 1 | −0.100 | 0.233 |

**How to read it (the mechanism, step by step):**
1. **Score each factor.** Compute 12-week momentum and 10/40 trend for each asset.
2. **Put on one axis.** Cross-sectionally z-score each (mean ~0 across the 3 assets).
3. **Merge → technical.** `0.5·z_mom + 0.5·z_trend`. SPY highest (+1.03), TLT lowest (−0.59).
4. **Rank.** SPY rank 3 (best), GLD 2, TLT 1 (worst).
5. **Allocate benchmark-relative weights.** Benchmark = 1/3 = 33.3%. Best asset gets
   +max_tilt (+10%), worst gets −10%, middle ~0. SPY 43.3% (overweight), TLT 23.3%
   (underweight = **synthetic short**, no borrow needed), GLD 33.3%.
6. Fully invested (weights sum to 1). Long/short is just over/underweight vs benchmark.

To regenerate with live numbers: `python examples/worked_example.py`.
